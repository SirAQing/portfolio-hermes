# WeKnora 项目结构文档

> 本文档梳理 WeKnora 项目的完整目录结构、技术栈、入口文件与架构模式，帮助开发者快速理解项目组织方式。
>
> 项目根目录：`E:\proCode\6Y\26-WeKnora`，核心代码位于 `WeKnora-main/` 子目录。

---

## 1. 项目概述

WeKnora 是腾讯开源的企业级 LLM 知识框架（v0.6.3，MIT 许可证），基于 RAG（检索增强生成）、ReAct Agent 和自动 Wiki 模式，将文档转化为可查询、可推理的知识资产。项目采用多语言 Monorepo 组织，包含 Go 后端、Vue 3 前端、Python 文档解析服务、CLI 工具、桌面应用和微信小程序。

## 3. 目录职责说明

| `mcp-server/` | Python MCP Server：Model Context Protocol 工具暴露 |

## 6. 架构概述

WeKnora 采用 **DDD 分层模块化单体 + 多语言 Monorepo + 微服务式部署** 的混合架构。

### DDD 分层架构（Domain-Driven Design）

后端 `internal/` 严格遵循领域驱动设计分层：

- **表现层**（`handler/`）：Gin HTTP 处理器，DTO 转换，SSE 流处理
- **应用层**（`application/service/`）：业务逻辑编排，60+ 服务文件
- **领域层**（`types/` + `types/interfaces/`）：80+ 实体定义 + 服务接口约定（依赖倒置）
- **基础设施层**（`infrastructure/` + `models/` + `application/repository/`）：分块器、文档解析、网页搜索、AI 模型适配、9 种向量数据库 retriever

### 依赖注入容器

`internal/container/container.go`（57KB）使用 `go.uber.org/dig` 构建完整 DI 容器，入口 `cmd/server/main.go` 中 `container.BuildContainer()` 统一组装所有依赖。`engine_factory.go` 以工厂模式创建向量检索引擎。

### 模块化单体

单一 Go 二进制，但 `internal/` 下 25 个独立子包，职责边界清晰。可切换组件通过接口 + 工厂模式实现：8 种向量库、7 种对象存储、7 种 IM 渠道、7 种网页搜索引擎、15+ LLM Provider。

### 多语言 Monorepo

所有代码在同一 Git 仓库，包含 4 个 Go 模块（主模块 + cli + client + desktop）、1 个 TypeScript/Vue 前端、2 个独立 Python 项目（docreader + mcp-server），通过 docker-compose 编排为多容器部署。

### 微服务式部署

`docker-compose.yml` 定义 23 个独立服务，核心常驻服务（app、frontend、docreader、postgres、redis）+ opt-in 向量库/对象存储/搜索/可观测服务（含 Langfuse 链路追踪栈 5 个服务、MCP Server 等）。服务间通过 gRPC（docreader）和 HTTP（mcp → app）通信。Helm Chart 支持 Kubernetes 生产部署。

### 多租户 RBAC

4 级角色矩阵（Owner / Admin / Contributor / Viewer），每租户审计日志、每资源所有权、跨租户超管，AES-256-GCM 凭证加密。

## 7. 文件放置约定

| 文件类型 | 放置位置 | 说明 |
|----------|----------|------|
| Go 源码 | `internal/<领域>/` | 按领域分包，handler → service → repository 分层 |
| Go 入口 | `cmd/<组件>/main.go` | 每个可执行组件一个 main.go |

| 前端状态 | `frontend/src/stores/` | 10 个 Pinia store |
| API 调用 | `frontend/src/api/` | 15+ API 子模块，对应后端 handler |

| Docker 构建 | `docker/Dockerfile.*` | 按组件分文件 |
| 运维脚本 | `scripts/*.sh` | 启动/构建/迁移脚本 |

## 8. 扩展与演进

### 添加新的向量数据库后端

在 `internal/application/repository/retriever/` 下新建子包，实现 retriever 接口，在 `internal/container/engine_factory.go` 注册工厂方法。

### 添加新的 IM 渠道

在 `internal/im/` 下新建子包，实现统一 adapter 接口（长连接管理 + QA 队列），在 DI 容器注册。

### 添加新的 Agent 工具

在 `internal/agent/tools/` 下新建 `.go` 文件，实现工具接口，在工具注册表中声明。

### 添加前端页面

在 `frontend/src/views/` 下新建模块目录，在 `frontend/src/router/` 注册路由，在 `frontend/src/api/` 添加对应 API 调用。

### 添加 CLI 命令

在 `cli/cmd/<命令名>/` 下新建 `.go` 文件，遵循 Options + Narrow Service 模式，在 `cmd/root.go` 注册。
